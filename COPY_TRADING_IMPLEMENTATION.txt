1. FEATURE OVERVIEW
=================================================================

The Copy Trading system allows users to follow and automatically copy the trades of successful crypto traders. This implementation includes both Mock Trading (practice with virtual funds) and Active Copy Trading (real funds).

IMPORTANT CONSTRAINT:
- Users can ONLY view EXECUTED trades
- Live/ongoing trades are HIDDEN for privacy and strategy protection
- This prevents front-running and maintains trader strategy confidentiality

4.0.1 FAVORITES TABLE
---------------------
CREATE TABLE favorites (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  trader_id integer NOT NULL,
  created_at timestamptz DEFAULT now(),
  UNIQUE(user_id, trader_id)
);

RLS Policies:
- Users can read own favorites
- Users can insert own favorites
- Users can delete own favorites

4.0.2 USER_REWARDS TABLE
-------------------------
CREATE TABLE user_rewards (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  task_type text NOT NULL,
  amount numeric(10,2) NOT NULL DEFAULT 0,
  status text NOT NULL DEFAULT 'pending',
  trading_volume numeric(15,2),
  created_at timestamptz DEFAULT now(),
  claimed_at timestamptz,
  CHECK (status IN ('pending', 'claimed', 'expired'))
);

RLS Policies:
- Users can read own rewards
- Users can update own rewards
- Users can insert own rewards

  target_progress numeric(15,2) NOT NULL,
  completed boolean DEFAULT false,
  updated_at timestamptz DEFAULT now(),
  UNIQUE(user_id, task_id)

RLS Policies:
- Users can read own task progress

- Users can insert own task progress

4.0.4 REFERRAL_STATS TABLE

CREATE TABLE referral_stats (

4.5 RLS POLICIES
-----------------
Enable RLS on all tables and create policies:

-- Users can only view their own copied traders
CREATE POLICY "Users can view own copied traders"
  ON user_copied_traders FOR SELECT
  TO authenticated
  USING (auth.uid() = user_id);

-- Users can only view their own copied trades
CREATE POLICY "Users can view own trades"
  ON user_copied_trades FOR SELECT
  TO authenticated
  USING (auth.uid() = user_id);

-- All users can view traders list (public data)
CREATE POLICY "Anyone can view traders"
  ON traders FOR SELECT
  TO authenticated
  USING (true);

-- All users can view executed trades (public data)
CREATE POLICY "Anyone can view executed trades"
  ON executed_trades FOR SELECT
  TO authenticated
  USING (true);

6. KEY BUSINESS LOGIC
=================================================================

TRADE VISIBILITY RULES:
- ONLY show executed (closed) trades
- NEVER show live/ongoing positions
- This protects trader strategies
- Prevents front-running

MOCK VS REAL MODE:
- Mock mode: is_mock = true, uses virtual funds
- Real mode: is_mock = false, uses actual USDT balance
- Both use same trade execution data
- Mock doesn't affect real balance

COPY RATIO CALCULATION:
- User investment amount / Trader's AUM = copy ratio
- User position size = Trader position size × copy ratio
- User P&L = Trader P&L × copy ratio

STARTING A COPY:
1. User selects trader
2. User sets investment amount
3. System calculates copy ratio
4. All future executed trades are automatically copied
5. User sees trade results in their dashboard

STOPPING A COPY:
1. User clicks "Stop Copying"
2. System marks trader as inactive
3. No new trades are copied
4. Existing positions remain until closed
5. User can see historical performance
